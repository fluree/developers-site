# History Syntax

<div class="overview">
  History queries enable auditable time travel — you can inspect every assertion
  and retraction that happened to your data across any range of commits.
</div>

---

## Overview

In Fluree, every transaction creates a new commit with a monotonically increasing index (`t`). History queries let you see exactly what changed, when, and in what direction (asserted or retracted).

History queries run through the same `POST /v1/fluree/query` endpoint as regular queries. The presence of a `"to"` key switches the query into history mode. Instead of returning the current state of the data, Fluree returns a log of changes over the specified time range.

---

## Request Syntax

A history query uses `"from"` and `"to"` to define a time range, with an `@t:` suffix indicating the commit index:

| Key        | Required | Value                                                                                           |
| ---------- | -------- | ----------------------------------------------------------------------------------------------- |
| `@context` | no       | **object** &bull; namespace prefix map, same as any other query                                 |
| `from`     | yes      | **string** &bull; ledger name with `@t:N` suffix for the start commit, e.g. `"cookbook/base@t:1"` |
| `to`       | yes      | **string** &bull; ledger name with `@t:latest` or `@t:N` suffix for the end commit             |
| `select`   | yes      | **array** &bull; variables to project into the result tuples                                    |
| `where`    | yes      | **object** or **array** &bull; pattern with optional `@t` and `@op` metadata bindings           |
| `orderBy`  | no       | **string** &bull; variable to sort results by (e.g. `"?t"` for chronological order)             |

### Time range notation

- `@t:0` — the initial (empty) state of the ledger, before any transactions
- `@t:1` — the first commit
- `@t:N` — the Nth commit
- `@t:latest` — the most recent commit

### Metadata bindings in `where`

Within your `where` clause, you can bind history metadata alongside the value of a property using a nested object:

```json
{ "@id": "?s", "schema:name": { "@value": "?name", "@t": "?t", "@op": "?op" } }
```

| Metadata key | Bound variable | Description                                              |
| ------------ | -------------- | -------------------------------------------------------- |
| `@value`     | `?name`        | The actual property value at that point in history       |
| `@t`         | `?t`           | The commit index at which this change occurred           |
| `@op`        | `?op`          | Either `"assert"` (added) or `"retract"` (removed)      |


## Filtering by Entity

To scope history to a single entity, constrain `?s` to a specific IRI in your `where` clause:

```json
{
  "@context": {
    "ex": "http://example.org/",
    "schema": "http://schema.org/"
  },
  "from": "cookbook/base@t:1",
  "to": "cookbook/base@t:latest",
  "select": ["?name", "?t", "?op"],
  "where": [
    {
      "@id": "ex:andrew",
      "schema:name": { "@value": "?name", "@t": "?t", "@op": "?op" }
    }
  ],
  "orderBy": "?t"
}
```

This returns only the name history for `ex:andrew`:

```json
[
  ["Andrew Johnson", 1, "assert"]
]
```

---

## Filtering by Time Range

Narrow to a specific range of commits by adjusting the `@t:` suffixes:

```json
{
  "@context": { "schema": "http://schema.org/" },
  "from": "cookbook/base@t:2",
  "to": "cookbook/base@t:5",
  "select": ["?s", "?name", "?t", "?op"],
  "where": [
    {
      "@id": "?s",
      "schema:name": { "@value": "?name", "@t": "?t", "@op": "?op" }
    }
  ]
}
```

This returns only changes that occurred between commits 2 and 5.

---

## Tracking Retractions

Since `?op` is bound to `"assert"` or `"retract"`, you can track name changes over time. When someone's name is updated, you'll see a retraction of the old value and an assertion of the new value at the same `?t`:

```json
[
  ["ex:andrew", "Andrew Johnson", 1, "assert"],
  ["ex:andrew", "Andrew Johnson", 3, "retract"],
  ["ex:andrew", "Andy the Yeti", 3, "assert"]
]
```

</Admonition>
